Oracle PriorityQueue Javadoc

The Backing Array (Binary Heap)

In Java, a PriorityQueue is implemented as a binary heap stored inside a contiguous array. The binary tree is kept complete (all levels filled except possibly the last, which is filled left-to-right), allowing nodes to be mapped directly to array indices without pointers:

public class PriorityQueue<E> extends AbstractQueue<E> {
    // Backing array stores heap elements
    transient Object[] queue; 
    
    // Number of elements in the queue
    int size;
}

Binary Heap Math Formulas

For any node stored at index i in the zero-indexed array:

  • Left Child Index: 2 * i + 1
  • Right Child Index: 2 * i + 2
  • Parent Index: (i - 1) / 2 (rounded down)

Sift-Up and Sift-Down Algorithms

To preserve the heap property (parent $\le$ child for min-heap), the queue reorganizes elements using two core routines:

Sift-Up (Inserting Elements)

When you call offer(e), the new element is appended at the end of the array (index size). If it is smaller than its parent, it swaps places. This repeats until it reaches the root or is $\ge$ its parent:

private void siftUp(int k, E x) {
    while (k > 0) {
      int parent = (k - 1) >>> 1;
      Object e = queue[parent];
      if (comparator.compare(x, (E) e) >= 0)
        break;
      queue[k] = e; // Pull parent down
      k = parent;   // Move pointer up
    }
    queue[k] = x;
  }

Sift-Down (Extracting Elements)

When you call poll(), the root (index 0) is removed. The last element in the array is moved to index 0, and is sifted down by swapping with its smaller child. This repeats until it has no children or is $\le$ both children:

private void siftDown(int k, E x) {
    int half = size >>> 1; // Loop while k has at least one child
    while (k < half) {
      int child = (k << 1) + 1; // Left child index
      Object c = queue[child];
      int right = child + 1;
      // If right child exists and is smaller than left child, use right
      if (right < size && comparator.compare((E) c, (E) queue[right]) > 0)
        c = queue[child = right];
      if (comparator.compare(x, (E) c) <= 0)
        break;
      queue[k] = c; // Pull smaller child up
      k = child;    // Move pointer down
    }
    queue[k] = x;
  }

Max-Heap Custom Comparator

By default, PriorityQueue implements a min-heap. To build a max-heap, configure it with a reversing comparator:

// Method 1: Using Comparator helper
  PriorityQueue<Integer> maxHeap = new PriorityQueue<>(Comparator.reverseOrder());
  
  // Method 2: Custom Lambda
  PriorityQueue<Integer> maxHeap = new PriorityQueue<>((a, b) -> Integer.compare(b, a));

The Iterator Ordering Trap

The iterator returned by priorityQueue.iterator() is not guaranteed to yield elements in order. It simply scans the backing array in linear index order ([0], [1], [2], ...). To retrieve elements in sorted order, you must call poll() repeatedly until the queue is empty.
