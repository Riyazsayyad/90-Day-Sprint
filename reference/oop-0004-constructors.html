Java Constructors & Initialization Reference

Study Reference for Constructor Chaining & Block Execution Sequences

1. Constructor Key Rules

  • Name Match: Must exactly match the name of the class (case-sensitive).
  • No Return Type: Has no return type, not even void. Adding void turns it into a standard method.
  • Modifiers: Can use access modifiers (public, protected, private, package-private), but cannot be declared with static, final, abstract, or synchronized.

2. Initialization Blocks vs. Constructors

Feature Static Initializer Block Instance Initializer Block
Execution Trigger Loaded once per class load into the JVM. Runs every time an object is instantiated.
Execution Order Runs during class loading, before any object creation. Runs after super() returns, before child constructor body.
Access Scope Can only access static class variables and methods. Can access both static and instance members.

3. Constructor Chaining Constraints

  • this(): Invokes another constructor in the same class to reuse logic. Must be the first statement in the constructor.
  • super(): Invokes a parent class constructor. Must be the first statement in the constructor.
  • Exclusivity: Since both statements must occupy the first line, they are mutually exclusive. You can write either this() or super() inside a single constructor block, never both.

4. Inheritance Execution Checklist

When executing new Child() where Child extends Parent:


   Class Loading:
   [1] Static Block of Parent runs (if not loaded)
   [2] Static Block of Child runs (if not loaded)

   Object Creation:
   [3] Parent Instance variables & Initializers execute
   [4] Parent Constructor body runs
   [5] Child Instance variables & Initializers execute
   [6] Child Constructor body runs

Note: Instance variable initialization values (e.g. private int x = 10;) are compiled directly into the constructor prefix space, executing immediately after the super() call returns.
