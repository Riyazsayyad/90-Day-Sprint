toString() Cheat Sheet & Reference

Secure Logging Patterns, Bidirectional Relationships, and Tool Options

1. Implementation Guidelines

Rule Rationale & Behavior Example Format
Start with ClassName Provides immediate type identification in massive log search outputs. "User[...]"
Essential State Only Omit large collections, binary fields (like raw byte arrays), or heavy nested records to save heap allocation and log size. Exclude list fields. Keep IDs and enum statuses.
Null Safety Avoid raw operations on nullable object properties. Use Objects.toString() or ternary checks. Objects.toString(name)
Standard Syntax Use standard JSON or property layouts to allow regex-based parser matching in ELK/Splunk. "User[id=12, name='Riyaz']"

2. Security Checklist: Sensitive Information

To avoid compliance violations (GDPR, PCI-DSS, HIPAA), apply these rules for data classifications:

  • Exclude Completely: Password hashes, raw tokens, PINs, cryptographic keys.
    Lombok: Place @ToString.Exclude on the field definition.
  • Apply Masking: Credit card numbers, Social Security Numbers (SSN), full personal names/emails (if required by strict privacy configurations). 
    // Masking Credit Card Number
private String maskCreditCard(String cc) {
    return (cc == null || cc.length() < 4) ? "****" : "****-****-****-" + cc.substring(cc.length() - 4);
}

3. Hibernate / Bidirectional Relationship Protection

Naively calling toString() on bidirectional associations causes infinite loops and crashes with a StackOverflowError.
Mitigation checklist:

  1. **Exclude bidirectional list collections** from parent class toString().
  2. **Exclude the parent reference field** from child class toString().
  3. Do not call getters on Lazy-loaded collections or proxy fields inside toString() to avoid triggering LazyInitializationException.

4. Coding Templates

Java 7+ Objects.toString Utility

Method Signature Output Behavior if Null
Objects.toString(o) Returns the string literal "null".
Objects.toString(o, "DEFAULT") Returns the custom fallback string "DEFAULT".

Lombok Annotation Setup

import lombok.ToString;

@ToString(callSuper = true)
public class Customer extends Person {
    private int id;
    
    @ToString.Exclude
    private String rawCardPin; // Excluded from logging
}
