Node Structure

Inside Java's LinkedList, the list elements are wrapped in Node objects:

private static class Node<E> {
    E item;
    Node<E> next;
    Node<E> prev;

    Node(Node<E> prev, E element, Node<E> next) {
        this.item = element;
        this.next = next;
        this.prev = prev;
    }
}

Bidirectional Traversal Optimization

To access index i, the node(int index) method splits the search in half based on size:

Node<E> node(int index) {
    // If target index is in first half, scan forward from head
    if (index < (size >> 1)) {
        Node<E> x = first;
        for (int i = 0; i < index; i++)
            x = x.next;
        return x;
    } 
    // Otherwise, scan backward from tail
    else {
        Node<E> x = last;
        for (int i = size - 1; i > index; i--)
            x = x.prev;
        return x;
    }
}

ArrayList vs. LinkedList Complexity

Operation ArrayList LinkedList LinkedList Details
get(index) / set(index) O(1) O(n) Requires traversal of n/2 steps on average.
add(E e) (to end) O(1) Amortized O(1) Trivial update of the last pointer and appending node.
add(index, E) (insertion) O(n) O(n) search + O(1) link Finding the node takes O(n), but pointer rewiring is O(1).
remove(index) (deletion) O(n) O(n) search + O(1) unlink Finding the node takes O(n), unlinking pointers is O(1).
remove(Object) (by value) O(n) O(n) search + O(1) unlink Must search from head to find value match.

Memory Overhead

In a 64-bit JVM, a LinkedList is significantly less memory-efficient than ArrayList:

  • ArrayList: Stores references contiguously in a single array. Overhead is only the unused array capacity (references are 4 or 8 bytes).
  • LinkedList: Each element requires a Node object wrapper.
    • Object header: 12-16 bytes
    • Three reference pointers (item, next, prev): 12-24 bytes
    • Padding alignment: 4-8 bytes
    • Total node overhead: ~32 to 48 bytes per element (in addition to the data itself).
