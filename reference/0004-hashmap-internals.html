HashMap Internals

Oracle HashMap

Core structure

table[] ← Node<K,V>[] (default capacity 16, always power of 2) index: [0] [1] [2] [3] ... null Node→ null Node→→ Node Node ← collision chain

Node fields

hash  — spread hash (cached)
key, value
next  — next node in same bucket (linked list)

Index calculation

hash = key.hashCode() spread:  h ^ (h >>> 16)
index = (capacity - 1) & hash    // same as hash % capacity when capacity is power of 2

put() outcomes

  1. Bucket empty → insert node
  2. Same key (hash + equals) → replace value
  3. Different key, same bucket → append to chain (or tree)

get() flow

Compute index → if bucket empty return null → walk chain/tree comparing hash then equals.

Resize

threshold = capacity × loadFactor   // default LF = 0.75
when size > threshold → double capacity, rehash all entries

Default: threshold 12 (16 × 0.75). First resize on the 13th entry.

Java 8+ treeification

ConstantValueMeaning
TREEIFY_THRESHOLD8Chain length → convert to red-black tree
UNTREEIFY_THRESHOLD6Tree shrinks → back to linked list
MIN_TREEIFY_CAPACITY64Below this table size, resize instead of treeify

Performance

  • Ideal: O(1) get/put — one bucket, no chain walk
  • Collision chain: O(n) worst case (list) or O(log n) (tree)
  • Iteration: O(capacity + size) — touches empty buckets too
