Class vs. Object Reference Guide

Study Reference for Runtime Structure & Class Loader Stages

1. Memory Allocation Zones (High-Level)

The JVM manages three main spaces for executing program code and storing variables/objects:

  • Stack (Per Thread): Stores execution frames containing primitive local variables and object reference pointers. Quick, structured allocation/deallocation.
  • Heap (Shared): Stores all actual objects, including arrays and instance variables. Governed by the Garbage Collector.
  • Metaspace (Shared): Stores class schema information, static members, method bytecodes, constants pool, and the virtual method tables (vtables).

2. Object Header Structure

Every object allocated on the heap is prefixed with an 8-to-16-byte metadata header:

Component Size (64-bit JVM) Description
Mark Word 8 bytes (64 bits) Holds synchronization lock flags, identity hashcode, thread bias pointers, and GC generational age bits.
Klass Word 4 or 8 bytes Pointer referencing the shared class schema metadata in the Metaspace (can be compressed to 4 bytes using Compressed OOPs).

3. Reference Variables vs. Heap Objects

Aspect Reference Variable (Stack) Actual Object (Heap)
Stored Value Memory address (pointer) or null. Object header metadata + instance data payload.
Allocation Target Method local stack frame. Shared heap region.
Lifetime Bound to method execution scope. Reclaimed by Garbage Collector when unreachable.

4. Structural Sequences

Class Loading Phase (Once)

  1. Loading: Locates and reads the binary .class file into memory.
  2. Linking:
    • Verification: Ensures bytecode complies with JVM safety and security rules.
    • Preparation: Allocates default memory values for static fields.
    • Resolution: Resolves symbolic references to direct memory addresses.
  3. Initialization: Executes static blocks (e.g. static { ... }) and assigns actual values to static fields.

Instantiation Phase (On new)

  1. JVM allocates required object byte size in heap.
  2. Sets Mark Word and references Klass Word in object header.
  3. Fills all instance fields with primitive default values.
  4. Executes parent and child initialization routines (fields assign, instance blocks).
  5. Executes the constructor body and returns the pointer address to the caller variable.
