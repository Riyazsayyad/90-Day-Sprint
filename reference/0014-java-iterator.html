Java Iterator Reference

Oracle java.util.Iterator Javadoc

1. Core API Contract

public interface Iterator<E> {
    boolean hasNext(); // Checks if more elements exist
    E next();          // Returns next element, advances cursor
    default void remove() { throw new UnsupportedOperationException(); }
}

2. Cursor Internals (ArrayList vs. LinkedList)

Feature ArrayList.Itr LinkedList.ListItr
Cursor Tracker int cursor (index of next element to return) Node<E> nextNode (reference to next node)
Last Returned int lastRet (index of last returned element; -1 if removed) Node<E> lastReturned (reference to last node; null if removed)
next() logic Returns elementData[cursor], sets lastRet = cursor, increments cursor++. Returns nextNode.item, sets lastReturned = nextNode, moves nextNode = nextNode.next.

3. Fail-Fast Mechanism (modCount)

To prevent dirty reads and data inconsistency in a multithreaded or mutating environment, Java iterators are fail-fast.

  • The collection maintains a counter field: int modCount (incremented on every add/remove/clear structural modification).
  • When an iterator is created, it initializes: int expectedModCount = modCount;
  • Every call to next() or remove() checks: 
    final void checkForComodification() {
    if (modCount != expectedModCount)
        throw new ConcurrentModificationException();
}

4. Why iterator.remove() is Safe

If you call collection.remove(obj) while iterating, the collection's modCount increases, but the iterator's expectedModCount is not updated. The next call to iterator.next() detects the mismatch and throws a ConcurrentModificationException.

When you call iterator.remove(), the iterator performs the deletion on the backing collection, increments the collection's modCount, and then synchronizes its own tracker:

// Inside ArrayList.Itr.remove()
ArrayList.this.remove(lastRet);
cursor = lastRet;
lastRet = -1;
expectedModCount = modCount; // <-- Synced to prevent Exception!

5. Time Complexity Analysis

Operation ArrayList.Itr LinkedList.ListItr
hasNext() O(1) O(1)
next() O(1) O(1)
remove() O(n) (due to array element shifting) O(1) (relinking pointers of current node)

6. Key Interview Questions

  1. What is the difference between Iterator and Enumeration?
    Enumeration is legacy (JDK 1.0), read-only (no remove()), and is not fail-fast. Iterator (JDK 1.2) is fail-fast and supports safe removal.
  2. Can you iterate and write to a collection simultaneously?
    Only if using iterator.remove(), or if using a concurrent collection like CopyOnWriteArrayList or ConcurrentHashMap which use fail-safe iterators operating on a clone of the backing structure.
