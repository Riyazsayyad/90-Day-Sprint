ArrayList Internals Reference

Oracle ArrayList Javadoc

Core Structure

Inside the JDK, an ArrayList is implemented as a wrapper around a primitive, dynamically-resized array:

// The array buffer into which the elements of the ArrayList are stored.
transient Object[] elementData;

// The size of the ArrayList (the number of elements it contains).
private int size;

Capacity vs Size

Metric Capacity Size
Definition The physical length of the backing array (elementData.length). The logical count of elements added by the user (size).
Default Starts at 0 (empty). Expands to 10 on the first insertion. Starts at 0.
Relation Always greater than or equal to Size (capacity >= size). Represents the index where the next element will be inserted.

Growth Strategy

When size == capacity, adding another element triggers a resize:

  1. Calculate new capacity (1.5x of the old capacity):
    newCapacity = oldCapacity + (oldCapacity >> 1)
  2. Allocate a new array of newCapacity.
  3. Copy all elements from the old array using System.arraycopy().
  4. Discard the old array (making it eligible for Garbage Collection).

Time Complexities

Operation Time Complexity Description / Cause
Random Access (get/set) O(1) Direct memory addressing math. No traversal needed.
Insertion (Add to End) O(1) Amortized Usually O(1). Occasional O(n) resize costs spread out over many additions.
Insertion (Arbitrary Index) O(n) Requires shifting all elements to the right from that index using System.arraycopy.
Deletion (Arbitrary Index) O(n) Requires shifting all elements to the left, and clearing the last element to avoid memory leaks.

Why is Random Access O(1)?

An array is allocated as a contiguous block of memory. To read the element at index i, the computer does not traverse the array. Instead, it computes the exact memory address directly in one math operation:

Address(i) = BaseAddress + i * ElementSize

This memory addressing takes constant time, meaning it is independent of the size of the array.
