The Backing TreeMap

Inside Java's TreeSet, elements are stored as keys in a backing TreeMap (which implements NavigableMap). The values are all bound to the dummy sentinel PRESENT:

public class TreeSet<E> extends AbstractSet<E> implements NavigableSet<E>, Cloneable, java.io.Serializable {
    private transient NavigableMap<E,Object> m;

    private static final Object PRESENT = new Object();

    // Natural ordering constructor
    public TreeSet() {
        this(new TreeMap<>());
    }

    // Custom ordering constructor
    public TreeSet(Comparator<? super E> comparator) {
        this(new TreeMap<>(comparator));
    }
    
    TreeSet(NavigableMap<E,Object> m) {
        this.m = m;
    }
}

Comparable vs. Comparator

To insert and sort elements, TreeSet requires an ordering rule. If neither is available, it throws a ClassCastException on insertion.

Property Comparable (Natural Ordering) Comparator (Custom Ordering)
Interface java.lang.Comparable<T> java.util.Comparator<T>
Method int compareTo(T o) int compare(T o1, T o2)
Location Implemented on the element class itself. Implemented in a separate helper class/lambda.
Usage Defined once for a class's default sorting. Allows multiple custom sort layouts.
Example String sorted lexicographically. Sort String by length: (s1, s2) -> Integer.compare(s1.length(), s2.length()).

The Duplication Gotcha

Unlike HashSet, which relies on hashCode() and equals() to check for duplicates, a TreeSet relies entirely on the comparison methods:

  • If compareTo(e) == 0 (or compare(a, b) == 0), the elements are considered equal/duplicates.
  • If this occurs, the new element is rejected (not added to the set).
  • Note: A set can violate the standard Set contract if its comparison is inconsistent with equals(). If equals() says two objects are different but compareTo() returns 0, the TreeSet will reject the second object as a duplicate!

Complexities & Ranges

A Red-Black Tree is a self-balancing binary search tree. Height is bounded by $2 \log(n + 1)$, ensuring fast lookups.

Operation TreeSet HashSet ArrayList / LinkedList
add / remove O(log n) guaranteed O(1) average / O(log n) worst O(n) (due to shifting or search)
contains O(log n) guaranteed O(1) average / O(log n) worst O(n) (linear search)

Range Query Operations

Because the tree is sorted, it supports efficient sub-range extraction:

  • first() / last(): Retrieve the minimum / maximum element.
  • higher(e) / lower(e): Retrieve the next greater / next smaller element.
  • subSet(fromElement, toElement): Return elements ranging from fromElement to toElement.
