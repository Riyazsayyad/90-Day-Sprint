Java ListIterator Reference

Oracle java.util.ListIterator Javadoc

1. Interface Methods

public interface ListIterator<E> extends Iterator<E> {
    // 1. Forward Traversal
    boolean hasNext();
    E next();
    int nextIndex();

    // 2. Backward Traversal
    boolean hasPrevious();
    E previous();
    int previousIndex();

    // 3. Structural Modifications
    void add(E e);     // Inserts before cursor position
    void remove();     // Deletes last returned item
    void set(E e);      // Replaces last returned item
}

2. Visualizing Cursor Positions

The iterator cursor always lies **between** elements, or before the first element, or after the last element. Below is a list of size 4:

   Index:       0          1          2          3
             [Element]  [Element]  [Element]  [Element]
   Cursor:   ^          ^          ^          ^          ^
            Start     Index 1    Index 2    Index 3     End
  • Calling next() returns the element to the right of the cursor and advances it.
  • Calling previous() returns the element to the left of the cursor and moves it back.

3. State Mutation Rules

Operation Targets Which Element? Under What Condition Allowed?
add(e) Inserts element **immediately before** the cursor. Always allowed. Resets lastRet = -1 (prevents immediate remove()/set()).
remove() Removes the element returned by the last next() or previous(). Only if neither add() nor remove() has been called since the last next()/previous().
set(e) Replaces the element returned by the last next() or previous(). Only if neither add() nor remove() has been called since the last next()/previous().

4. Iterator vs. ListIterator

Feature Iterator ListIterator
Traversal Direction Forward only. Bi-directional (forward and backward).
Modifications Supported remove() only. remove(), set(), and add().
Index Querying No index trackers. Supports nextIndex() and previousIndex().
Target Structures Any Collection (List, Set, Queue). Only indexable structures implementing List.
