java.lang.Object Cheat Sheet & Reference

Core Contracts, JVM Memory Structures, and Synchronization States

1. Memory Footprint of an Object

Header Component Size (64-bit JVM) Information Tracked
Mark Word 8 bytes (64 bits) Identity Hashcode (25 bits), GC age (4 bits), Biased lock flag (1 bit), Lock state flags (2 bits).
Klass Word 8 bytes (4 if compressed) Pointer to Class Metadata in Metaspace (enabling `getClass()` and type checks).
Instance Data Variable Primitive values and object references declared in class hierarchy.
Padding 0 to 7 bytes Aligns total object size to a multiple of 8 bytes for 64-bit alignment performance.

2. equals() and hashCode() Contract Checklist

To implement proper equality checks, ensure your class complies with the following:

  • Symmetry: If a.equals(b), then b.equals(a). Be careful when comparing base classes and subclasses.
  • Consistent HashCode: If you override equals(), you **must** override hashCode() to avoid breaking collections.
  • Lombok/Records Alternative: Prefer Java records to auto-generate proper implementations.
// Safe Boilerplate implementation using java.util.Objects
@Override
public boolean equals(Object o) {
    if (this == o) return true;
    if (o == null || getClass() != o.getClass()) return false;
    User user = (User) o;
    return Objects.equals(id, user.id) && Objects.equals(name, user.name);
}

@Override
public int hashCode() {
    return Objects.hash(id, name);
}

4. Core Object Methods Summary

Method Signature Modifier Behavior and Warnings
boolean equals(Object obj) Virtual Compares references by default. Must override with hashCode() consistently.
int hashCode() Virtual native Returns a pseudo-random value cached in object header. Must align with equals contract.
String toString() Virtual Returns string description. Override to assist logging. Prevent PII leaks.
Class getClass() Final native Reads Klass Word pointer. Primary entry point for Reflection. Cannot be overridden.
Object clone() Protected native Shallow copy. Avoid using. Prefer copy constructors/records.
void finalize() Protected DEPRECATED. Do not use. Use Try-With-Resources or Cleaners.
void wait(), notify(), notifyAll() Final (native) Coordinators for synchronization. Must own object monitor to call.
