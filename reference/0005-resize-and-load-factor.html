Resize & Load Factor

Oracle HashMap

Formulas

threshold = capacity × loadFactor
resize when:  size > threshold
new capacity = old capacity × 2   (always power of 2)

Defaults

ConstantValue
DEFAULT_INITIAL_CAPACITY16
DEFAULT_LOAD_FACTOR0.75
First threshold12
First resize at13th entry

Tradeoffs

Load factorEffect
Higher (e.g. 0.9)Less memory, longer chains, slower lookup
Lower (e.g. 0.5)More memory, shorter chains, faster lookup
0.75 (default)JDK's balanced default

Rehash on resize

Index mask grows by one bit. Each entry stays at oldIndex or moves to oldIndex + oldCapacity.

Avoiding resize

new HashMap<>(expectedMappings);  // Java 19+
new HashMap<>((int)(n / 0.75f) + 1);  // older JDKs
