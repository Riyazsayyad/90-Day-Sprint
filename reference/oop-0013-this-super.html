this & super Cheat Sheet

Constructor Chaining Invariants, Resolution Matrix, and Thread Safety Guards

1. Context & Scope Matrix

Keyword Instance Context (Non-Static) Class Context (Static) Typical Use Case
this Allowed. Points to the current executing heap instance. Forbidden. Causes compiler error. Resolve variable shadowing, Builder patterns.
super Allowed. Refers to parent class layer components. Forbidden. Causes compiler error. Access hidden fields or execute parent overridden methods.

2. Constructor Chaining Rules

  • Chaining Placement: this() or super() must be the first line of the constructor body.
  • Mutual Exclusion: You cannot place both this() and super() in the same constructor.
  • Compiler Default behavior: If neither is declared, compiler inserts an implicit no-argument super() call.
  • Safety Guarantee: Initializes state top-down (Object down to child subclass) to prevent child logic from accessing uninitialized parent state.

3. Field Hiding vs. Method Overriding

Feature Fields (Variables) Methods
Resolution Type Static (Compile-time) Dynamic (Runtime lookup)
Determinant The **Reference Type** of the variable. The actual **Concrete Object type** on heap.
Hiding/Overriding Subclass field hides parent field. Both exist in memory. Subclass method overrides parent method. Parent method bypassed.
External Access Access hidden parent via cast: ((Parent) child).field. Cannot bypass overridden method from external calls.

4. Critical Lifecycle Safety Guards

1. No Overridable Method Calls in Constructors

Calling an overridable method in a constructor triggers it dynamically via the subclass before the subclass constructor has run. The overridden method runs against uninitialized subclass fields (which are still default/null), causing NullPointerExceptions or corrupt initial states.

2. No "this" escaping during Construction

Do not publish or share the this reference from inside a constructor (e.g., registering to a global listener or starting a thread). Other threads will obtain access to a partially-constructed object, violating thread safety and memory visibility guarantees.

3. Spring AOP Proxy bypass

Inside a Spring bean, calling this.method() bypasses transactional AOP interceptors because the invocation does not pass through the bean's surrounding proxy object. Autowire a self-reference or use AspectJ compiler integration if inner calls need transaction guards.

5. Quick Code Reference

class Base {
    int id = 1;
    void show() { System.out.println("Base"); }
}

class Sub extends Base {
    int id = 2; // Hides Base.id
    
    @Override
    void show() { 
        super.show(); // 1. Calls Base.show()
        System.out.println("Sub id: " + id); // 2. Sub.id (2)
        System.out.println("Base id: " + super.id); // 3. Base.id (1)
    }
}
