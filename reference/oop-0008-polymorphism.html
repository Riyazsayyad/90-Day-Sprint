Java Polymorphism & Binding Reference

Study Reference for Binding Types, Reference Casting, and Pattern Matching Syntax

1. Overloading vs. Overriding Contracts

Feature Method Overloading (Static) Method Overriding (Dynamic)
Resolution Compile time (early binding). Runtime (late binding).
Method Name Must be identical. Must be identical.
Parameters Must be different (count or types). Must be exactly identical.
Return Type Can be different. Must be identical or covariant.
Exceptions Can throw different exceptions. Cannot expand checked exceptions list.

2. Dynamic Dispatch Architecture (vtable lookup)


    Stack Frame                Heap Memory                   Metaspace (Class Metadata)
  +-------------+          +------------------+             +--------------------------+
  |  gateway   | ------>  |   StripeGateway  |             | StripeGateway.class      |
  | (Reference) |          |------------------|             |--------------------------|
  |  [Type:     |          |  [Klass Word]    | ----------> |  [vtable]                |
  |  Payment-   |          |  [Instance Data] |             |  - processPayment() ---->| Ptr to Stripe's
  |  Gateway]   |          +------------------+             |  - toString()            | implementation
  +-------------+                                           +--------------------------+

During compilation, the compiler only verifies that the reference type (PaymentGateway) contains the method. At runtime, the JVM uses the heap object's Klass Word header reference to locate the class vtable in Metaspace and execute the overridden method address.

3. Reference Casting Rules

  • Upcasting (Safe & Implicit): 
    NotificationSender sender = new EmailSender(); // Implicitly upcast
    Allows uniform treatment of subclass instances. You can only call methods defined in the NotificationSender interface.
  • Downcasting (Explicit & Conditional): 
    // UNSAFE: Throws ClassCastException if sender is not EmailSender
EmailSender email = (EmailSender) sender; 

// SAFE (Traditional):
if (sender instanceof EmailSender) {
    EmailSender email = (EmailSender) sender;
    email.sendHtmlEmail();
}
  • Modern Pattern Matching (Java 16+): 
    // SAFE (Modern):
if (sender instanceof EmailSender email) {
    email.sendHtmlEmail(); // Variable 'email' is auto-scoped
}

4. Core Framework Patterns Utilizing Polymorphism

  • Dependency Injection (DI): Client classes depend on interfaces, while the framework injections supply concrete implementations at runtime, allowing adapters swapping without breaking core logic.
  • Template Method Pattern: Framework classes define basic algorithm structures (e.g. Spring's JdbcTemplate), letting developers override specific hooks to inject custom queries.
