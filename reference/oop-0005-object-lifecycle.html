2. Reference Strength Types Comparison

Reference Type GC Collection Condition Primary Use Cases & Characteristics
Strong Never collected while reachable. Standard variables. Hard memory references.
Soft Collected only when OutOfMemoryError is imminent. Memory-sensitive cache buffers. Reclaimed to prevent OOM.
Weak Collected during the next GC cycle. Metadata tables (e.g. WeakHashMap). Cleared immediately when strong references drop.
Phantom Collected after object is finalized/freed. Post-mortem resource cleanup. Combined with ReferenceQueue. get() always returns null.

3. Resource Cleanup Alternatives

Resource cleanup is critical in backend systems. The table below details modern alternatives to finalizers:

Characteristic finalize() Method (Deprecated) AutoCloseable & try-with-resources
Control JVM / Garbage Collector decides timing. Developer manages scope; execution is deterministic.
Performance High overhead (adds objects to queue; extra GC cycle). Extremely low overhead. Incurs standard method call execution costs.
Security Vulnerable to Finalizer Attacks (object resurrection). Secure. State is closed and exceptions are handled safely.
