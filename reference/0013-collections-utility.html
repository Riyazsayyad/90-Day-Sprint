Oracle java.util.Collections Javadoc

Wrapper Collections (Views vs. Copies)

The java.util.Collections class provides wrapper methods that return views of a backing collection. They intercept calls to delegate or block operations, rather than copying any internal array or node data structure.

Wrapper Type Creation Method Behavior & Constraints
Unmodifiable unmodifiableList(list)
unmodifiableSet(set)
unmodifiableMap(map)		 Blocks writing calls by throwing UnsupportedOperationException. Read-through views: changes to the underlying backing list are visible.
Synchronized synchronizedList(list)
synchronizedSet(set)
synchronizedMap(map)		 Wraps all reads and writes in a synchronized(mutex) block. Caveat: Manual synchronization on the mutex is required when iterating.
Checked checkedList(list, Class)
checkedSet(set, Class)
checkedMap(map, K, V)		 Enforces runtime type verification on item insertions. Prevents heap pollution when dealing with legacy non-generic code.

Core Static Algorithms

  • sort(List<T> list): Sorts the specified list using natural ordering. Delegates to list.sort(null), which uses Timsort (hybrid merge/insertion sort) with O(n log n) time complexity and O(n) space complexity.
  • binarySearch(List<? extends Comparable> list, T key): Searches the list using a binary search algorithm.
    • If the list implements RandomAccess (like ArrayList), execution is O(log n).
    • If the list is a sequential-access chain (like LinkedList), execution degrades to O(n) link traversal costs.
  • shuffle(List<?> list): Randomly permutes list elements using a Fisher-Yates shuffle in O(n).
  • reverse(List<?> list): Reverses element order in O(n) time.

Immutable Collections vs. Unmodifiable Wrappers

Feature Unmodifiable View (e.g. Collections.unmodifiableList) Immutable Collection (e.g. List.of, List.copyOf)
Underlying Mutations Reflected (view reads directly from the backing mutable list). Truly immutable (backing data cannot change; copies on creation).
Null Elements Permitted if the backing collection permits them. Prohibited (throws NullPointerException on creation/lookup).
