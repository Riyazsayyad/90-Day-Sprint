Lambda Expressions

~7 minutes · builds on functional interfaces

What a lambda is

A lambda expression is a compact way to supply the single abstract method of a functional interface — without naming the method or the class.

// Anonymous inner class (5+ lines)
Comparator<String> byLen = new Comparator<String>() {
    public int compare(String a, String b) {
        return a.length() - b.length();
    }
};

// Lambda (1 line)
Comparator<String> byLen = (a, b) -> a.length() - b.length();

Source: Oracle — Lambda Quick Start

Syntax — three parts

(parameters) -> body Argument list · arrow token · expression or block

Parameters

  • Zero: () -> 42
  • One (type inferred): x -> x * 2 — parentheses optional
  • One (typed): (int x) -> x * 2
  • Many: (a, b) -> a + b — parentheses required

Types are usually inferred from context (target typing). If you type one parameter, type them all.

Source: Oracle — Syntax of Lambda Expressions

Body — expression vs block

// Expression body — result is returned automatically
Function<Integer, Integer> square = n -> n * n;

// Block body — use return for non-void results
Function<Integer, Integer> square = n -> {
    int result = n * n;
    return result;
};

// Void — single statement needs no braces
Consumer<String> print = s -> System.out.println(s);

A block body that must produce a value needs an explicit return on every path. A block with only s.isEmpty() does not return a boolean — it just calls the method and discards the result.

Target typing

Lambdas have no standalone type. The compiler checks them against the expected functional interface:

Predicate<String> isBlank = s -> s.isBlank();
//  ^ target type tells compiler: test(String) -> boolean

That is why a lambda alone — s -> s.isEmpty() — does not compile. It needs a target.

Practice — pick the valid lambda

Target type: Predicate<String> (method boolean test(String)).

Which lambda is valid?

Retrieval — rewrite from memory

Cover the answer, try yourself first — that effort is what builds retention.
