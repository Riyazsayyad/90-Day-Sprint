Java Iterator Internals: Interview Guide

~8 minutes · ties to mission

1. Why Iterator Was Introduced

Prior to Java 1.2, traversing collections was inconsistent. Legacy structures like Vector used Enumeration. Iterator was introduced to achieve two main goals:

  • Standardization: Provides a uniform traversal interface across all collection types (Lists, Sets, Maps), hiding the underlying structural layout (Encapsulation).
  • Safety: Supports safe structural modifications (removal) during iteration and implements fail-fast checks to detect concurrent modifications.

2. Internal Cursor Mechanism

Each collection class implements Iterable and defines an internal private class (e.g., ArrayList.Itr or LinkedList.ListItr) implementing the Iterator interface. They maintain internal pointers to keep track of state:

  • ArrayList Iterator: Tracks int cursor (index of the next element to return) and int lastRet (index of the last element returned; reset to -1 after removal).
  • LinkedList Iterator: Tracks a Node reference nextNode (the next node to visit) and lastReturned (the last node visited; reset to null after removal).

3. Methods Internals (hasNext, next, remove)

The standard loop checks and traverses elements sequentially:

// hasNext() checks if cursor hasn't reached size limit
public boolean hasNext() {
    return cursor != size;
}

// next() retrieves the element and increments the cursor
public E next() {
    checkForComodification(); // Mismatch check
    int i = cursor;
    if (i >= size) throw new NoSuchElementException();
    Object[] elementData = ArrayList.this.elementData;
    cursor = i + 1;
    return (E) elementData[lastRet = i];
}

4. The Fail-Fast vs. Safe-Remove Mechanics

Why does collection.remove() throw a ConcurrentModificationException while iterator.remove() does not? It comes down to two variables:

  • modCount: Maintained by the collection, counting structural modifications.
  • expectedModCount: Maintained by the iterator, initialized to the collection's modCount at startup.

If you call list.remove(), only modCount increments. The next call to iterator.next() notices that modCount != expectedModCount and triggers a fail-fast crash.

If you call iterator.remove(), it performs the removal, increments the collection's modCount, and immediately re-synchronizes the iterator's state:

public void remove() {
    if (lastRet < 0) throw new IllegalStateException();
    checkForComodification();
    try {
        ArrayList.this.remove(lastRet);
        cursor = lastRet;
        lastRet = -1;
        expectedModCount = modCount; // <-- STATE RE-SYNCED!
    } catch (IndexOutOfBoundsException ex) {
        throw new ConcurrentModificationException();
    }
}

5. Time Complexity Analysis

  • hasNext() and next(): O(1) across both List structures.
  • iterator.remove():
    • ArrayList: O(n). Although finding the node is O(1), deleting it requires shifting array elements via System.arraycopy().
    • LinkedList: O(1). The node pointer is held directly by the iterator, meaning relinking the previous and next nodes requires only constant time.

6. Key Interview Questions

Q: What happens if you call remove() twice in a row without calling next()?
It throws IllegalStateException. After the first call to remove(), the iterator resets lastRet = -1. The second call checks if (lastRet < 0) and fails immediately because you cannot remove an element that hasn't been retrieved yet.
Q: What is the difference between fail-fast and fail-safe iterators?
Fail-fast: Throws ConcurrentModificationException on structural modifications during iteration (e.g., ArrayList, HashSet).
Fail-safe (Weakly Consistent): Iterates over a clone of the collection or handles concurrent changes smoothly without throwing exceptions (e.g., CopyOnWriteArrayList, ConcurrentHashMap).

7. Real-World Use Cases

  • Filtering Collections: Removing items matching a predicate during traversal (equivalent to modern collection.removeIf(filter)).
  • Custom Collections: Implementing the Iterable interface in your custom structures to allow users to leverage the standard foreach loop syntax.
