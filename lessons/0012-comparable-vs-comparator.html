Comparable vs. Comparator

~6 minutes · ties to mission

The Core Difference

In Java, sorting and ordering require clear rules. Java provides two interfaces for this, each with a different intent:

Comparable (Natural Ordering) Implemented by the class itself to define its default sorting behavior (e.g., Numbers sorted numerically, Strings sorted alphabetically).
Method: public int compareTo(T o)
Comparator (Custom/External Ordering) Implemented externally (often as a lambda) to sort objects using alternative rules, or to sort classes you do not own/modify.
Method: public int compare(T o1, T o2)

The Return Contract

Both methods follow a strict mathematical return contract. They return an integer value indicating the order of elements:

// For Comparable (compareTo)
int result = obj1.compareTo(obj2);

// For Comparator (compare)
int result = comp.compare(obj1, obj2);
  • Negative Integer: obj1 is smaller/comes before obj2.
  • Zero (0): obj1 is equal to obj2.
  • Positive Integer: obj1 is larger/comes after obj2.

Under the Hood: TreeSet & Duplicate Rejection

This is where collection internals become critical. Unlike HashSet (which uses hashCode() and equals()), TreeSet relies entirely on comparisons.

When you insert an element into a TreeSet (backed by TreeMap):

  1. Java traverses the Red-Black tree, comparing the new element to the current node.
  2. If the comparison returns a negative number, it walks left. If positive, it walks right.
  3. If the comparison returns 0, **the TreeSet rejects the new element as a duplicate**, and does not modify the set.
Warning: Uniqueness Gotcha If your comparison logic returns 0 for two objects, they are duplicates in the eyes of a TreeSet, even if equals() says they are different. A TreeSet can violate the standard Set contract if its comparison is inconsistent with equals().

Let's See It in Code

Below is a custom class implementing Comparable for natural ordering, paired with a custom Comparator lambda:

import java.util.*;

class User implements Comparable<User> {
    int id;
    String name;

    User(int id, String name) {
        this.id = id;
        this.name = name;
    }

    // Natural order: sort by id
    @Override
    public int compareTo(User other) {
        return Integer.compare(this.id, other.id);
    }
}

public class Main {
    public static void main(String[] args) {
        // TreeSet using Natural Order (Comparable)
        Set<User> byId = new TreeSet<>();
        byId.add(new User(10, "Riyaz"));
        byId.add(new User(5, "Sayyad"));
        // Reject: duplicate ID (10) even with a different name!
        byId.add(new User(10, "Alice")); 

        // TreeSet using Custom Order (Comparator)
        Set<User> byNameLength = new TreeSet<>(
            (u1, u2) -> Integer.compare(u1.name.length(), u2.name.length())
        );
        byNameLength.add(new User(1, "Riyaz"));   // length 5
        byNameLength.add(new User(2, "Sayyad"));  // length 6
        // Reject: length 5 is a duplicate of "Riyaz"!
        byNameLength.add(new User(3, "Alice"));   
    }
}
