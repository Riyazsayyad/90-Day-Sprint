Predicate, Function, Consumer, Supplier

~8 minutes · the four shapes you'll use most

One sentence each

Java ships 43 functional interfaces in java.util.function. Four cover almost everything:

InterfaceShapeMethod
Predicate<T>T → booleantest
Function<T,R>T → Rapply
Consumer<T>T → voidaccept
Supplier<T>() → Tget

Source: Oracle — java.util.function

Predicate — test, don't transform

Answers: "Should I keep this?"

Predicate<Integer> isEven = n -> n % 2 == 0;

List.of(1, 2, 3, 4).stream()
    .filter(isEven)   // filter expects Predicate
    .forEach(System.out::println);  // 2, 4

filter drops elements where test returns false.

Function — transform

Answers: "What does this become?"

Function<String, Integer> length = String::length;

List.of("hi", "hello").stream()
    .map(length)      // map expects Function
    .forEach(System.out::println);  // 2, 5

Input type T and output type R can differ — that's the point of the second type parameter.

Consumer — act, return nothing

Answers: "Do something with this." Side effects live here — printing, logging, saving.

Consumer<String> print = System.out::println;

List.of("a", "b").forEach(print);  // forEach expects Consumer

Source: Consumer javadoc — "expected to operate via side-effects"

Supplier — produce on demand

Answers: "Give me a value." No input — lazy production.

Supplier<String> id = () -> UUID.randomUUID().toString();

System.out.println(id.get());  // new UUID each call
System.out.println(id.get());

Common pattern: defer expensive work until someone calls get().

See all four in one pipeline

List<Integer> nums = List.of(10, 15, 20, 25);

nums.stream()
    .filter(n -> n % 2 == 0)    // Predicate — keep evens
    .map(n -> n * n)            // Function — square
    .forEach(System.out::println);  // Consumer — print (400)
