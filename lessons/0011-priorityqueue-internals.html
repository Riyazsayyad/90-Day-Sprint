PriorityQueue Internals

~12 minutes · builds on ArrayList internals

The Backing Array (Binary Heap)

In Java, a PriorityQueue is structured under the hood as a binary heap. A binary heap is a complete binary tree (all levels are filled completely, except possibly the bottom level, which is filled from left to right).

Because the tree is guaranteed to be complete, it can be packed into a flat, contiguous primitive array (represented as Object[] queue) without the need for left/right node reference pointers. This merges the structural speed of tree hierarchies with the memory efficiency of contiguous arrays.

Source: Oracle PriorityQueue API Specifications

Mapping Trees to Array Indexes

By filling tree nodes level-by-level, we establish a mathematical link between tree parent-child relations and array indices. For any element located at index i in the zero-indexed array:

  • Left child index is calculated as: 2 * i + 1
  • Right child index is calculated as: 2 * i + 2
  • Parent index is calculated as: (i - 1) / 2 (rounded down)

For example, if parent node is at index 0, its left child is at 2(0)+1 = 1, and its right child is at 2(0)+2 = 2. If a node is at index 4, its parent is at (4-1)/2 = 1.

Sift-Up and Sift-Down Operations

To preserve the min-heap ordering constraint (that each parent node must be $\le$ both of its child nodes), the priority queue implements two index-relocating algorithms:

  • Sift-Up (Inserting): When adding an element via offer(e), the value is appended at the end of the array (bottom-right of tree). It is compared to its parent; if smaller, they swap. This process repeats upward until the heap property is satisfied or the element becomes the root.
  • Sift-Down (Extracting): When pulling the minimum element via poll(), the root (index 0) is returned. The last element in the array is relocated to index 0, and is sifted downward by swapping with its smaller child. This repeats until it has no children or is smaller than both.

Max-Heaps using Comparators

By default, Java's PriorityQueue sorts elements in their natural ascending order (acting as a min-heap). If you need a max-heap (where poll() retrieves the largest element first), you must initialize it with a custom comparator that reverses the natural ordering:

PriorityQueue<Integer> maxHeap = new PriorityQueue<>(Comparator.reverseOrder());

Under the hood, the sift-up and sift-down algorithms will compare elements using comparator.compare(x, parent). If the comparator reverses the check, swaps will happen when children are larger than parents, turning the structure into a max-heap.

Time Complexities

  • peek(): O(1). The highest priority item is always at index 0 of the backing array.
  • offer() / poll(): O(log n). Restoring the heap constraint requires traversing a single branch of the tree, which is bounded by the logarithmic tree height.
  • Heapify constructor (new PriorityQueue(collection)): O(n). Building a heap bottom-up is a standard algorithm that runs in linear time.

Retrieval Practice

Cover the answer. If you iterate through a PriorityQueue using an iterator or a enhanced for-each loop, does it yield elements in sorted ascending order?

Reveal

No, it does not. The iterator simply scans the backing array sequentially from index 0 to size - 1. While index 0 is guaranteed to be the minimum, the remaining array indexes are arranged to satisfy parent-child heap constraints, not linear sorted order. To extract elements in sorted order, you must call poll() in a loop until the queue is empty.
