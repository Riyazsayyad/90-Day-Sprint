Java ListIterator Internals: Interview Guide

~8 minutes · ties to mission

1. Why ListIterator Exists

The standard Iterator interface is designed to work across all Collections (including Sets and Queues). Because of this, it is limited to forward-only traversal and supports only element deletion.

Lists are index-ordered collections. To exploit this indexable structure, Java introduced the specialized ListIterator (which extends Iterator) to support:

  • Bi-directional traversal (moving backward and forward).
  • Modifying elements during iteration using set().
  • Inserting elements mid-stream using add().
  • Querying element index positions without advancing the cursor.

2. Forward and Backward Traversal

In addition to next() and hasNext(), ListIterator adds:

  • hasPrevious(): Returns true if there are elements behind the cursor.
  • previous(): Returns the element behind the cursor and moves the cursor one position backward.
  • nextIndex() / previousIndex(): Returns the index of the element that would be returned by a subsequent call to next() or previous().

3. Visualizing Cursor Positions

The core concept to master for interviews is that **the cursor always lies in the gaps between elements** (before the first, between two, or after the last element):

   Element:     [Element 0]  |  [Element 1]  |  [Element 2]
   Index:            0       |       1       |       2
   Cursor:      ^            ^               ^             ^
              Start       Between         Between         End
             Index 0     0 and 1         1 and 2        Index 3

For example, if the cursor is at the Start (before index 0):

  1. Calling next() returns Element 0. The cursor shifts right (lies between 0 and 1).
  2. Calling previous() returns Element 0. The cursor shifts left back to the Start.
Interview Tip: Iteration Flip If you call previous() immediately after next(), you will retrieve the **exact same element** you just read!

4. State Modifications: add(), remove(), and set()

The behavior of modifications depends on the last traversed element:

  • add(e): Inserts e immediately **before the cursor** position. The cursor shifts right so the new element is behind it. Subsequent calls to next() are unaffected, but calling previous() returns this newly added element.
    Constraint: Calling add() resets the last-returned index tracker (lastRet = -1). You cannot call remove() or set() immediately after adding.
  • remove(): Deletes the last element returned by next() or previous().
    Constraint: Can only be called once per call to next()/previous(), and cannot be called immediately after add().
  • set(e): Replaces the last element returned by next() or previous() with e.
    Constraint: Cannot be called immediately after add() or remove().

5. Internal Implementation (ArrayList vs. LinkedList)

  • ArrayList: ListItr is an inner class extending Itr. It implements previous() by decrementing the internal cursor index: 
    public E previous() {
    checkForComodification();
    int i = cursor - 1;
    if (i < 0) throw new NoSuchElementException();
    Object[] elementData = ArrayList.this.elementData;
    cursor = i;
    return (E) elementData[lastRet = i];
}
  • LinkedList: ListItr maintains node references. Since LinkedList is a doubly-linked list, calling previous() is trivial and fast: it simply reads nextNode.prev.

6. Common Interview Questions

Q: What happens if you call listIterator.add() followed by listIterator.remove()?
It throws IllegalStateException. In the OpenJDK implementation, calling add() clears the state pointer by setting lastRet = -1. Since remove() requires lastRet >= 0, it fails immediately to prevent state corruption.
Q: What is the difference between Iterator and ListIterator?
Iterator is uni-directional, operates on any collection, and supports read-and-delete. ListIterator is bi-directional, restricted to List implementations, supports index queries, and allows insertions/replacements during traversal.
