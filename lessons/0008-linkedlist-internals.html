LinkedList Internals

~9 minutes · builds on ArrayList internals

The Node Structure

Unlike ArrayList, which aggregates its objects inside a contiguous backing array, a LinkedList stores its elements inside individual, disjoint heap-allocated objects called nodes.

In Java, java.util.LinkedList implements a doubly linked list. Each node is an instance of a private static nested class containing three fields:

private static class Node<E> {
    E item;         // The actual data element
    Node<E> next;   // Reference to the next node in the list
    Node<E> prev;   // Reference to the previous node in the list
}

The list itself maintains three basic references: int size, Node<E> first (pointing to the head), and Node<E> last (pointing to the tail).

Why Random Access is O(n)

Because nodes are allocated dynamically in separate segments of memory, they do not occupy a contiguous physical block. This means we cannot calculate a node's physical address directly from an index using simple arithmetic.

To find the element at index i, the JVM must perform a list traversal starting at one end of the list and following the chain of pointers step-by-step:

Node<E> x = first;
for (int k = 0; k < index; k++) {
    x = x.next;
}
return x.item;

Since the number of hops is proportional to the index value, lookup takes linear O(n) time.

Bidirectional Optimization

As a doubly linked list, Java optimizes this traversal. If the target index is in the second half of the list (index > size / 2), it starts from the last pointer and walks backward using prev. While this cuts search steps in half, the worst-case complexity remains O(n).

O(1) Insertions (When Reference is Known)

Adding an element to an ArrayList requires shifting subsequent items to the right to make room (taking O(n) time). A LinkedList never shifts items. If you already hold a direct reference to a node (say, target), inserting a new node involves only re-linking four pointer fields:

  1. Create the new node: Node<E> newNode = new Node<>(target, element, target.next);
  2. Update the following node: target.next.prev = newNode;
  3. Update the target node: target.next = newNode;

Because this re-linking relies on a fixed number of variable updates, it takes constant O(1) time. (Note: Locating the node by index first still requires an O(n) search, but the insertion operation itself is O(1) once you are at that node).

Doubly Linked List Trade-Offs

The extra pointers in a doubly linked list add both advantages and costs:

  • Pros: Efficient traversal in both directions, simple insertion and deletion at both ends (first and last) in O(1) time, which makes it ideal for implementing double-ended queues (Deques).
  • Cons: High memory overhead. Every node must store two object pointers (prev and next references) in addition to the element reference. In a 64-bit JVM, this pointer metadata can easily consume more memory than the actual payload data.

Retrieval Practice

Cover the answer. If you are adding elements exclusively to the end of a list using add(E element), which structure is generally faster and uses less memory: ArrayList or LinkedList?

Reveal

ArrayList is generally faster and uses significantly less memory. An ArrayList packs references contiguously without node wrappers, reducing metadata overhead. Adding to the end is an amortized O(1) operation, which runs extremely fast due to sequential memory access patterns, whereas LinkedList must allocate a new node object on every single insertion.
